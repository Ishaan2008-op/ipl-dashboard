# IPL Dashboard (Python + Streamlit)

A mini project that analyzes IPL match and ball-by-ball data using **Pandas** and **Seaborn/Matplotlib**, and can be displayed as an interactive dashboard using **Streamlit**.

---

## What’s inside this repo

- **project.py**: the main analysis script (data loading, grouping, metrics, and plots).
- **\*.png**: plots generated by the script (saved using `plt.savefig(...)`).

> Note: There is no `requirements.txt` yet in the repo, so the README includes the common install commands you likely used. If you add a `requirements.txt`, you can replace the install steps with `pip install -r requirements.txt`.

---

## Commands used (and why)

This section explains the commands you typically run to set up and execute this project, plus why each command is needed.

### 1) Clone the repository

```bash
git clone https://github.com/Ishaan2008-op/ipl-dashboard.git
cd ipl-dashboard
```

**Why:**
- `git clone` downloads the project to your system.
- `cd` moves into the project folder so you can run Python/Streamlit commands from the right location.

---

### 2) Create and activate a virtual environment (recommended)

#### Windows (PowerShell)
```bash
py -m venv .venv
.\.venv\Scripts\Activate.ps1
```

#### macOS / Linux
```bash
python3 -m venv .venv
source .venv/bin/activate
```

**Why:**
- Keeps project dependencies isolated from your global Python installation.
- Avoids version conflicts when you install libraries like Pandas/Streamlit.

---

### 3) Install required libraries

```bash
pip install pandas numpy matplotlib seaborn streamlit
```

**Why these packages are used:**
- **pandas**: load CSV files (`pd.read_csv`) and perform grouping/aggregation (`groupby`, `sum`, `count`).
- **numpy**: numeric utilities (imported in `project.py`; useful for calculations and arrays).
- **matplotlib**: figure control and saving plots (`plt.figure`, `plt.savefig`, `plt.show`).
- **seaborn**: high-level statistical charts (`sns.barplot`, `sns.boxplot`, `sns.stripplot`, `sns.scatterplot`, `sns.heatmap`, `sns.histplot`).
- **streamlit**: (imported) for building an interactive dashboard UI if you choose to run it with Streamlit.

---

### 4) Run the analysis script (generates plots)

```bash
python project.py
```

**Why:**
- Executes the full analysis pipeline in `project.py`.
- Generates and saves images such as `top_scorers_plot.png`, `top_strike_rates_plot.png`, `top_wicket_takers_plot.png`, and the strike-rate visualizations.

---

### 5) Run as a Streamlit app (optional)

If you want this project to run as a dashboard, you usually run:

```bash
streamlit run project.py
```

**Why:**
- Launches Streamlit’s local web server and opens the app in your browser.
- This is the standard way to run any Streamlit-based dashboard.

> Important: `project.py` currently mainly uses Matplotlib/Seaborn and `print(...)`. To make it a full Streamlit dashboard, you would replace `print(...)` with `st.write(...)` and display figures using `st.pyplot(fig)` (or `st.image(...)` for saved PNGs).

## What the code is doing (high level)

Inside `project.py`, the main steps are:

1. **Load data** with `pd.read_csv`.
2. **Top run scorers**: `groupby('batsman')` + `sum()` + `sort_values()` and then `sns.barplot(...)`.
3. **Strike rate**: runs/balls * 100, then visualize top strike rates.
4. **Extra strike-rate visualizations** (inside a `try:` block):
   - Boxplot + stripplot (per-innings strike rate spread)
   - Scatter (strike rate vs balls faced)
   - Heatmap (avg strike rate by season, if season exists)
   - Histogram/KDE distribution
5. **Top wicket takers**: filter dismissals with `.isin([...])`, then `groupby('bowler')` + `count()`.
6. **Economy rate**: compute runs conceded excluding byes/legbyes; count legal balls; convert balls→overs; compute economy; plot best/worst.
